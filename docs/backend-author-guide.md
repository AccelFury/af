# Backend Author Guide

Backend crates expose capabilities, probe tools, prepare command specs, and
return structured reports. Backends must not call shells by default.

Minimum backend checklist:

- declare a stable backend id;
- expose capabilities and whether each is supported or planned;
- probe external tools with `program + args`;
- write or return expected artifacts;
- report commands, versions, warnings, and limitations;
- keep vendor tools optional for default CI.

Generated backend scripts must be marked `Generated by AccelFury IP Toolchain`.
