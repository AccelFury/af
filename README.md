# AccelFury FPGA IP Monorepo

`af` is the unified AccelFury FPGA/IP monorepo. It combines the Rust CLI,
manifest/schema tooling, reusable RTL core packages, board registry, toolchain
templates, vector generation, reports, and CI checks.

Current scope:

- parse and validate `af-core.toml`;
- validate core structure and declared RTL files;
- scaffold SystemVerilog or Verilog-2001 starter cores;
- run Verilator lint/smoke checks when Verilator is available;
- export a FuseSoC `.core`;
- write JSON and Markdown reports;
- generate GitHub Actions CI;
- maintain board/toolchain registries imported from `core-template`;
- ship `cores/af-mod-add` and `examples/af-pdm-rx` as first packaged cores.

## Quick Start

```bash
cargo run -p af-cli --bin af -- doctor --json
cargo run -p af-cli --bin af -- manifest validate examples/af-pdm-rx/af-core.toml
cargo run -p af-cli --bin af -- manifest validate cores/af-mod-add/af-core.toml
cargo run -p af-cli --bin af -- core check examples/af-pdm-rx
cargo run -p af-cli --bin af -- core new /tmp/af-demo --name af-demo --language verilog
cargo run -p af-cli --bin af -- registry check
cargo run -p af-cli --bin af -- board matrix --output docs/board_matrix.md
cargo run -p af-cli --bin af -- wrapper generate examples/af-pdm-rx --target fusesoc
```

`af core lint examples/af-pdm-rx --backend verilator` runs `verilator --lint-only` if Verilator is installed. If it is not installed, the command returns a structured `AF_BACKEND_UNAVAILABLE` error.

## Generated Outputs

By default, generated files are written under `.af-build/`:

- `.af-build/reports/*.json`
- `.af-build/reports/*.md`
- `.af-build/fusesoc/*.core`

Every generated artifact is marked with `Generated by AccelFury IP Toolchain`. JSON reports keep that marker in the `generated_by` field so they remain valid JSON.

Buyer-grade FPGA IP work should keep `af-core.toml`, integration wrappers,
CI, report artifacts, and release claims aligned. Missing lifecycle capability
belongs in [TODO.md](TODO.md) or in a direct `af` code change.
